01 平台在做的事 · The Flywheel

一条知识飞轮，把资料变成资产、再用反馈回炼

平台的全部功能，都挂在这条闭环上。每个环节都有专属页面、专属服务、专属权限。

01

资料库

导入原始资料为不可变版本，内容哈希去重，版本链可追溯。

02

资产化

五阶段流水线 + LLM 抽取，把资料编译成知识资产包与组件。

03

证据补全

每个组件回链到源资料版本，形成可核查的证据记录。

04

审核标注

质量门禁衍生人工任务，团队审核、标注、修正。

05

版本发布

跑质量门禁、冻结 manifest 快照，发布不可变版本给 Agent。

06

Agent 反馈

Agent 通过 MCP 消费知识，命中/未命中反馈回流，驱动下一轮。

资料 → 资产 → 证据 → 审核 → 发布 → 消费 → 反馈回到资料，闭环自增强 ↻

02 平台怎么搭的 · The Architecture

前后端同仓，路由 → 服务 → 数据的薄分层

TypeScript 全栈：Fastify 同时承担 JSON API 与构建后 SPA 托管；另有独立 MCP 服务器把已发布知识暴露给外部 Agent。

前端SPA
React 19 + React Query，按子域拆页：Sources Assets Review Release AgentFeedback Diagnostics。组件不直接 fetch，统一走 API 层。
接口Fastify
JWT / CORS / multipart、全链路 tracing、统一错误处理。入参先过 zod schema 再进服务；写操作用角色门禁（denyRole / requireRole）。
业务Services
薄业务层，按子域拆分；路由不直接拼 SQL，依赖注入容器统一装配。读模型聚合与写路径（导入 / 构建 / 发布）各自独立。
存储PostgreSQL
连接池 + 自动迁移（CREATE / ALTER IF NOT EXISTS 幂等）+ seed。行映射集中走 mappers，JSONB 列统一兼容映射。上传 blob 与构建工作区落本地数据目录。
消费MCP Server
独立 stdio 服务器，复用查询服务暴露 kb_* 工具。只读「当前发布版本」的冻结快照，不读 draft 资产。

04 平台的脊梁 · The Data Spine

可信的根基：三条不可变内容链

平台「可审计」的底气，来自内容快照不可变——重导入走新版本，发布冻结 manifest，绝不原地改写。名称/备注等元数据才可改。

资料链 不可变
source_blobs（哈希去重）→ source_bundle_versions（parent 链）→ source_files
资产链 可重生
knowledge_build_runs（状态机）→ asset_packages → asset_components → evidence_records
治理链 流转
quality_gate_profiles · knowledge_rule_profiles → review_tasks → 标注修正
发布链 不可变
releases（冻结 manifest）→ release_channels（当前指针）→ Agent 只读快照
反馈链 回流
agent_events · mcp_audit · attribution_audits → 驱动下一轮迭代
05 价值出口 · The Consumption

知识的唯一出口：Agent 通过 MCP 消费

所有沉淀，最终是为了让 Agent 用得准、用得稳。工具只读「当前发布版本」的冻结快照——保证 Agent 看到的永远是经过审核、可复现的知识。

// 两条入口，一份实现

HTTP + stdio

  • HTTP：POST /api/mcp/query，JWT 保护，写 mcp_audit 审计
  • stdio：标准 MCP 协议，外部 Agent 直接接入
  • 两者共用同一份查询服务，行为一致
// kb_* 工具集

结构化知识查询

  • kb_search / kb_get_page / kb_get_entity — 检索页面与实体
  • kb_query_table / kb_validate_table — 查询与校验表格数据
  • 每次调用带证据与可信度，Agent 知道「这条知识有多可信」
06 这对你意味着什么 · Why It Matters

开发者得到确定性，产品得到信任

// FOR ENGINEERS

给开发者

  • 清晰的「路由→服务→数据」分层，新增能力有固定落点
  • 内容链不可变 + 迁移幂等，schema 演进安全
  • 强类型 + Zod 校验入参，领域类型集中导出
  • 行映射集中、JSONB 统一兼容，读写路径可预测
  • 全链路 tracing + 结构化日志，问题可按 trace 复盘
// FOR PRODUCT

给产品

  • 资料到资产全程可追溯，每条知识都有证据与可信度
  • 发布不可变、可复现，Agent 永远消费经审核的版本
  • 质量门禁 + 人工审核双关卡，低质知识发不出去
  • Agent 使用反馈自动回流，知识持续迭代增强
  • 「可审计、可管理、可发布」贯穿每个环节
